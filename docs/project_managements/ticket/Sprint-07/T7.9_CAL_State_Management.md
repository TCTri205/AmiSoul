## 2. Mô tả tác vụ
Xây dựng logic quản lý trạng thái cho các thực thể trong Contextual Awareness Layer (CAL), đảm bảo các sự kiện có thời hạn và các trạng thái dở dang được cập nhật đúng tiến độ thời gian thực.

## 3. Tiêu chí chấp nhận (Chi tiết)
- [ ] Triển khai **CALStateManager** để điều phối vòng đời của `Active_Expectations` và `Pending_States`.
- [ ] **Fast-track Sync Support:** Xây dựng API nội bộ để Stage 4 có thể ghi nhanh các thực thể CAL mới vào Redis L1.
- [ ] Định nghĩa các trạng thái cho sự kiện: `Active` -> `Expired_Unacknowledged` (12h) -> `Archived`.
- [ ] Logic xử lý khi User quay lại:
    - Nếu sự kiện là `Active` (đang diễn ra): Kích hoạt `Time_Anomaly` flag.
    - Nếu sự kiện là `Expired_Unacknowledged`: Đẩy vào context để AI thực hiện Proactive Follow-up.
- [ ] Đồng bộ hóa trạng thái giữa Redis L1 (tốc độ cao) và PostgreSQL L2 (lưu trữ lâu dài) qua Consolidation Job.
- [ ] Đảm bảo cơ chế quét (Cleanup/Transition) không gây tải cho Database.
