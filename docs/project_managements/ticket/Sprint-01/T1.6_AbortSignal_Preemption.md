## 2. Mô tả tác vụ
Triển khai cơ chế Preemption (Hủy ngang) sử dụng `AbortController` để dừng các tiến trình xử lý cũ khi người dùng gửi tin nhắn mới trong lúc AI đang "suy nghĩ".

## 3. Tiêu chí chấp nhận (Chi tiết)
- [x] Thiết lập cấu trúc `AbortSignal` truyền từ Gateway qua Stage 1 đến Stage 4.
- [x] Khi nhận tin nhắn mới vào Buffer (Stage 0), gọi `abort()` cho pipeline đang chạy.
- [x] Triển khai **Preemption Limiter**: Tối đa 2 lần preempt liên tiếp, sau đó chuyển sang Batch Mode.
- [x] **Emotional Flooding Bypass**: Nếu `Session_Vibe` cực đoan, bỏ qua limiter để phản hồi tức thời (Fast Acknowledge).
- [x] Kiểm tra và xử lý ngoại lệ `AbortError` tại các điểm gọi API bên thứ ba (Gemini API).
- [x] Đảm bảo tài nguyên (Connection, Memory) được giải phóng ngay lập tức sau khi ngắt dòng.
