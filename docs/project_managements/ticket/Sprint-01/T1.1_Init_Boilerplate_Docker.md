## 2. Mô tả tác vụ
Thiết lập môi trường phát triển cơ bản bằng cách khởi tạo project NestJS và cấu hình Docker.

> [!NOTE]
> **Chiến lược Hybrid Development:** Dự án sử dụng Docker chỉ để quản lý Database (PostgreSQL) và Cache (Redis). Đối với NestJS, khuyến khích chạy trực tiếp trên máy host (`npm run dev`) trong quá trình phát triển để tiết kiệm tài nguyên (RAM/CPU) và giảm độ trễ khi build. Dockerfile Multi-stage chỉ dùng để đóng gói triển khai (Staging/Production).


## 3. Tiêu chí chấp nhận (Chi tiết)
- [ ] Khởi tạo thành công dự án NestJS với TypeScript version mới nhất.
- [ ] Cấu hình ESLint và Prettier theo chuẩn dự án (Airbnb/Prettier).
- [ ] Viết `Dockerfile` sử dụng cơ chế Multi-stage (Base, Build, Production) để tối ưu kích thước image.
- [ ] Thiết lập file `.env.example` với đầy đủ các biến môi trường (DB_URL, REDIS_URL, GEMINI_API_KEY).
- [ ] Code được review và pass các bài kiểm tra linting/formatting.
