## 2. Mô tả tác vụ
Xây dựng dịch vụ trung tâm để truy xuất dữ liệu ngữ cảnh từ nhiều nguồn khác nhau (Redis cho dữ liệu nóng, PostgreSQL cho ký ức dài hạn).

## 3. Tiêu chí chấp nhận (Chi tiết)
- [ ] Triển khai `ContextRetrieverService` trong NestJS.
- [ ] Thực hiện **Affective Retrieval** (Truy xuất song song) từ:
    - Redis (CAL L1): Các sự kiện đang diễn ra, trạng thái mong đợi.
    - PostgreSQL (CMA/CAL L2): Ký ức episodic sử dụng pgvector.
- [ ] Triển khai thuật toán xếp hạng: `Score = (0.5 * Vector_Sim) + (0.3 * Affective_Align) + (0.2 * Recency)`.
- [ ] Áp dụng **Bonding Filter**: Giới hạn độ sâu truy xuất dựa trên `Bonding_Score`.
- [ ] Đảm bảo độ trễ truy xuất tổng thể < 100ms.
- [ ] Xử lý lỗi (Error Handling) nếu một trong các nguồn dữ liệu không khả dụng.
