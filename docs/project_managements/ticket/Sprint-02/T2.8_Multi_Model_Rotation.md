## 2. Mô tả tác vụ

Triển khai một lớp trừu tượng (Abstraction Layer) cho các dịch vụ AI, cho phép hệ thống sử dụng đồng thời nhiều nhà cung cấp (Groq, Gemini) và nhiều API Key với cơ chế xoay vòng ưu tiên (Priority Rotation). Điều này giúp tối ưu hóa chi phí, tăng giới hạn Rate Limit và đảm bảo tính sẵn sàng cao.

## 3. Tiêu chí chấp nhận (Chi tiết)

- [x] **Unified Interface:** Thiết lập `LlmProvider` interface chung (hỗ trợ cả Chat Simulation và Embedding) cho tất cả các dịch vụ AI.
- [x] **Multi-Provider Support:**
    - Triển khai `GroqProvider` sử dụng Groq API.
    - Triển khai `GeminiProvider` sử dụng Google Generative AI SDK.
- [x] **Priority Rotation Logic:**
    - Mặc định sử dụng **Groq** làm Priority 1 (Tốc độ cao, chi phí thấp).
    - Tự động fallback sang **Gemini** nếu Groq hết quota hoặc lỗi.
- [x] **API Key Rotation:**
    - Hỗ trợ danh sách nhiều API Key cho mỗi Provider (đọc từ `.env` hoặc config).
    - Cơ chế **Round-robin** hoặc **Failover** để chuyển sang key tiếp theo khi nhận mã lỗi `429 (Too Many Requests)`.
- [x] **Error Handling:**
    - Nhận diện và xử lý chính xác các mã lỗi đặc thù của từng Provider.
    - Tích hợp Circuit Breaker ở cấp độ Provider.
- [x] **Integration:**
    - Cập nhật `PerceptionService` và `SimulationService` để sử dụng Provider Orchestrator mới này thay vì gọi trực tiếp SDK.
