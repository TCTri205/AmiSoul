## 2. Mô tả tác vụ
Triển khai logic Backend để chuyển đổi từ tài khoản Guest (vô danh, định danh qua Device_ID) sang tài khoản chính thức (Email/Social) mà không làm mất dữ liệu ký ức và mức độ thân thiết.

## 3. Tiêu chí chấp nhận (Chi tiết)
- [ ] **OTP Service:** Triển khai API gửi và xác thực OTP qua Email (sử dụng Nodemailer hoặc mock service cho MVP).
- [ ] **Account Linking Logic:**
  - API nhận `Device_ID` và `Email_Auth_Token`.
  - Thực hiện "Merge" dữ liệu: Chuyển toàn bộ `Memories`, `Bonding_Score`, `Session_Logs` từ bản ghi Guest sang bản ghi User chính thức trong PostgreSQL.
  - Xóa hoặc vô hiệu hóa bản ghi Guest cũ để tránh trùng lặp.
- [ ] **JWT Refresh:** Cấp lại JWT mới với `UserID` chính thức sau khi link thành công.
- [ ] Đảm bảo tính toàn vẹn dữ liệu (Data Integrity) bằng Database Transaction.
