## 2. Mô tả tác vụ
Triển khai tính năng gửi dữ liệu giọng nói/âm thanh và xử lý giao diện UI phức tạp khi người dùng ngắt ngang AI đang streaming (Preemption).

## 3. Tiêu chí chấp nhận (Chi tiết)
- [x] **Voice UI (Media Input):**
  - [x] Trong `InputArea`, thêm nút Microphone.
  - [x] Sử dụng `MediaRecorder API` để thu âm. Khi giữ/chạm thu âm, hiển thị Waveform Animation nhỏ (có thể dùng CSS scale pulse).
  - [x] Kết thúc thu âm, gửi audio dạng `Blob` (WebM/MP4) qua sự kiện `socket.emit('user_audio')`.
- [x] **Image Input (Media Input):**
  - [x] Trong `InputArea`, thêm nút 📷 (ẩn trong menu "+").
  - [x] Chọn ảnh -> hiển thị preview thumbnail trong Input Area.
  - [x] Gửi ảnh dạng `Base64` hoặc `Blob` qua sự kiện `socket.emit('user_image')`.
- [x] **Preemption UI (Ngắt ngang luồng):**
  - [x] Khi user gửi tin nhắn/âm thanh MỚI trong khi luồng AI đang stream.
  - [x] Phát sự kiện `socket.emit('interrupt')`.
  - [x] Bubble hiện tại của AI phải lập tức dừng hiệu ứng typing (cursor biến mất).
  - [x] Áp dụng hiệu ứng **Fade-out 50% opacity** ở phần cuối cùng của text AI, và chèn thêm ký tự `...` để biểu thị bị cắt ngang.
  - [x] Sau > 2 lần ngắt ngang liên tiếp, nếu nhận `batch_mode_start` từ Backend, hiển thị Notification nhẹ: "Ami đang đọc hết tin của bạn...".
- [x] Tích hợp Plugin React Markdown cho **AI Action Rendering**:
  - [x] Dùng Regex tìm các cụm `*action*` (ví dụ `*mỉm cười*`) do LLM sinh ra.
  - [x] Bóc tách và render chúng thành `ActionTag` inline: font size 0.75em, italic, màu nhạt (`text-muted`).
- [x] **Reaction UI:**
  - [x] Hỗ trợ long-press (mobile) hoặc hover (desktop) trên tin nhắn AI để hiện thanh reaction (❤️, 😢, 😡, 😂, 👍).
  - [x] Phát sự kiện `socket.emit('message_reaction', { messageId, emoji })`.
  - [x] Hiển thị reaction icon nhỏ ở góc bubble.
