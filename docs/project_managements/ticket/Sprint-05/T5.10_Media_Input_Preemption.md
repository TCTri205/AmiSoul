## 2. Mô tả tác vụ
Triển khai tính năng gửi dữ liệu giọng nói/âm thanh và xử lý giao diện UI phức tạp khi người dùng ngắt ngang AI đang streaming (Preemption).

## 3. Tiêu chí chấp nhận (Chi tiết)
- [ ] **Voice UI (Media Input):**
  - Trong `InputArea`, thêm nút Microphone.
  - Sử dụng `MediaRecorder API` để thu âm. Khi giữ/chạm thu âm, hiển thị Waveform Animation nhỏ (có thể dùng CSS scale pulse).
  - Kết thúc thu âm, gửi audio dạng `Blob` (WebM/MP4) qua sự kiện `socket.emit('user_audio')`.
- [ ] **Preemption UI (Ngắt ngang luồng):**
  - Khi user gửi tin nhắn/âm thanh MỚI trong khi luồng AI đang stream.
  - Phát sự kiện `socket.emit('interrupt')`.
  - Bubble hiện tại của AI phải lập tức dừng hiệu ứng typing (cursor biến mất).
  - Áp dụng hiệu ứng **Fade-out 50% opacity** ở phần cuối cùng của text AI, và chèn thêm ký tự `...` để biểu thị bị cắt ngang.
  - Sau > 2 lần ngắt ngang liên tiếp, nếu nhận `batch_mode_start` từ Backend, hiển thị Notification nhẹ: "Ami đang đọc hết tin của bạn...".
- [ ] Tích hợp Plugin React Markdown cho **AI Action Rendering**:
  - Dùng Regex tìm các cụm `*action*` (ví dụ `*mỉm cười*`) do LLM sinh ra.
  - Bóc tách và render chúng thành `ActionTag` inline: font size 0.75em, italic, màu nhạt (`text-muted`).
