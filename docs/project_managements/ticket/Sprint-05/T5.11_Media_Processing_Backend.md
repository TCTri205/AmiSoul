## 2. Mô tả tác vụ
Triển khai logic xử lý dữ liệu đa phương tiện (Giọng nói và Hình ảnh) tại Stage 0 của Backend để chuyển đổi chúng thành văn bản mô tả trước khi đưa vào pipeline xử lý ngôn ngữ chính.

## 3. Tiêu chí chấp nhận (Chi tiết)
- [ ] **Speech-to-Text (STT) Service:**
  - Nhận sự kiện `user_audio` từ Socket.
  - Sử dụng Gemini Pro Vision hoặc OpenAI Whisper (tùy cấu hình) để convert audio sang text.
  - Text trích xuất được chèn vào Pipeline như một tin nhắn bình thường kèm metadata `origin: voice`.
- [ ] **Image Captioning Service:**
  - Nhận sự kiện `user_image` từ Socket.
  - Sử dụng Gemini Pro Vision để mô tả nội dung ảnh (Visual Understanding).
  - Text mô tả được bọc trong tag XML `<image_context>...</image_context>` và đưa vào Pipeline.
- [ ] **Error Handling:** Trả về event `media_error` nếu file hỏng hoặc không xử lý được.
- [ ] Tối ưu hóa tốc độ xử lý Media Stage 0 < 2s.
